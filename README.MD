## 接下来的任务

目前已经有了一些可用于rust的文档和通用指令。

接下来我希望再为它添加一些提示，把这些提示转为更合适、精准的表达写入到CLAUDE-appoint.md中合适的位置：
- 在每一次开始设计、开发、测试等行为之前，都要重新检阅@CLAUDE.md。
- 在每一次设计、开发、测试完成之后，都要把相关的变动写入到@CLAUDE.md中，
- 在项目开始时和每一次设计、开发、测试完成之后都必须：
    - 创建git提交。
    - 遵循[语义化版本](https://semver.mdn.cn/)维护changelog。
- 每一个新的功能开发、功能修补、功能测试等，都要在单独的git分支中进行开发，在该分支目标完成后再合并到主分支。
- 在遇到未知问题或非常模糊的情况时，必须明确表示“我不知道”，并询问用户。
