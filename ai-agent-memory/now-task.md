# 现在的任务

用户想在general目录下的现有指令体系中添加一些内容，请你分析内容后生成可实现用户目标的指令并将其添加到general下正确的文档位置中：

一共两个任务，

1. 是跟mcp调用规范相关的

我希望在每一次对`mcp-***`的mcp工具（例如mcp-time、mcp-web-search、mcp-desktop-commander、mcp-serena等）的调用之后都要把添加此次mcp调用相关信息记录到mcp-log.md，通常这个添加记录的行为本身也需要一个mcp-time调用和一个写入文件相关操作的mcp调用，所以应该是，除了这个写入操作本身引起的时间和文件写入这两次mcp调用之外的所有mcp调用，都必须要创建对应的mcp-log记录，

比如你要调用一次mcp-web-search，就应该是先调用mcp-time获取时间戳A，然后调用mcp-web-search，再马上把调用原因、参数、结果等需要记录的信息加上时间戳A，记录到mcp-log.md

同时，希望你改良一下记录格式，.md对于数据查看，少量还行，但重量或大量会很不便，我希望你改良一下记录格式和规范，使用csv格式进行记录，类似AI-AGENT_working-status.csv那种

2. 是跟任务前置准备工作相关的

要强化待定项处理机制的实际执行，必须在任务内容优化阶段进行全面的深度的分析，多方面多维度的衡量，思考可以优化的地方、用户可能没有考虑到的地方等，这一步要做的是列出所有优化的可能性，按照undetermined-template.md列出来，可能只是轻度优化、高度优化、极大性能提升，或是对于要实现的目标提出其他可行性建议、更换解决方法（最终做到一致或极近似的效果，但实现路径不同），也可以尝试超前优化、过度理解，最终交由用户决断即可

不论是有没有优化的可能性，有没有待优化项，都必须告知经过用户确认后才能继续后续环节，如果当前任务内容描述已经是最优解、最准确无歧义、最具有可行性、实现代价最小、综合评估最高效，则需要给出得到此“已是最优”的评估结果的详细依据，然后告知用户，待用户确认，

总之这个环节必须要用户确认后才能继续下一步
