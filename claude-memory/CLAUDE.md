# 任务内容前置准备

> !!!注意!!!
>
> **任务内容前置准备**中需要**询问用户**的地方，不要直接向用户询问，
>
> 要参考@undetermined-template.md中的格式，将待定项先写入到undetermined.md中，
>
> 完全思考完成之后，再一起向用户确认，
>
> 让用户把不采纳的待定项从undetermined.md中删除，把要采纳的具有多个优化的待定项仅保留采纳的优化，删除掉多余可能分支，
>
> 当用户确认后，检查undetermined.md是否每个待定项只有一个可能目的，如果还存在多个可能，要提醒用户，直到没有歧义，
>
> 然后把，所有最终保留下来的建议采纳的待定项或没被删除的建议去除的待定项的优化结果，写入到**本次任务内容**中，至此即为完成了询问和优化。

1. 必须全面审阅项目内容，分析项目情况
2. 深度思考@now-task.md中的**现在的任务**中的内容，进行优化：
   1. 描述是否准确，是否足够准确传达任务目的，足够得知要实现的效果
   2. 对于描述不够准确的地方，思考其可能的目的，然后列出这些可能性，询问用户
   3. 描述是否冗杂，对于描述冗杂的地方，应该：
      1. 思考，其可能是真的冗杂，可以直接删去
      2. 思考，其可能不是冗杂，而是用户的表达出现误差
      3. 根据上述可能性，列出描述冗杂的部分可能的目的，然后询问用户
3. 在分析之后更新具体的任务内容到下面的**本次任务内容**中，向用户确认**本次任务内容**是否准确无误，得到确认后再进行下一步，否则根据用户指示进行
4. 然后再按照**本次任务内容**开始完成本次任务

# 本次任务内容

对**任务内容前置准备**流程进行准确性优化，提高任务理解和执行的精确度
