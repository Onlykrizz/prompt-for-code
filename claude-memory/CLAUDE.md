# CLAUDE.md

## 核心流程指引

所有过程都应遵循@CLAUDE-appointment.md（开发任务）或@CLAUDE-appointment-general.md（通用任务）。

各类过程开始时、结束时都应检查@CLAUDE-check.md。

先完成**任务内容前置准备**。

## 任务内容前置准备

### 待定项处理机制

**重要说明**：任务准备阶段需要询问用户的地方，按以下流程处理：

1. 参考@undetermined-template.md格式，将待定项写入undetermined.md
2. 完整思考后统一向用户确认
3. 用户确认后删除不采纳的待定项，多方案待定项仅保留采纳方案
4. 确保每个保留待定项只有唯一实施方案
5. 将确认结果整合到**本次任务内容**

### 准备流程

1. **项目情况分析**
   - 根据任务复杂度审阅相关内容
   - 分析项目结构、现有规范、相关文档、类似任务处理方式

2. **任务内容优化**
   - **准确性分析**：检查任务描述准确性，识别歧义和不明确之处
   - **简洁性分析**：识别冗余部分，区分真实冗余与表达误差

3. **确认与执行**
   - 更新**本次任务内容**
   - 向用户确认后开始执行

### 异常处理和质量保证
- 用户不配合删除待定项时再次澄清确认
- 建立任务完成质量检查标准
- 提供回滚和重新分析机制
