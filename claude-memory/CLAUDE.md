# 任务内容前置准备

> **重要说明**
>
> **任务内容前置准备**中需要**询问用户**的地方，不要直接向用户询问，
>
> 要参考@undetermined-template.md中的格式，将待定项先写入到undetermined.md中，
>
> 完全思考完成之后，再一起向用户确认，
>
> 用户确认并删除不采纳的待定项，对于有多个方案的待定项仅保留采纳的方案，
>
> 当用户确认后，确认每个保留的待定项都只有唯一的实施方案，如发现同一待定项下仍有多个方案被保留，需要提醒用户进一步选择，直到没有歧义，
>
> 将用户确认的优化结果整合到**本次任务内容**中，完成询问和优化流程。

1. 根据任务复杂度，审阅相关项目内容，分析项目情况
   审阅内容包括：项目结构、现有规范、相关文档、类似任务的处理方式等

2. 深度思考@now-task.md中的**现在的任务**中的内容，进行优化：

   1. **准确性分析**：检查任务描述是否准确传达目的，识别歧义和不明确之处
      - 对于描述不够准确的地方，思考其可能的目的，然后列出这些可能性，询问用户
   2. **简洁性分析**：检查任务描述是否存在冗余，识别可以精简的部分
      - 评估是否为真正的冗余信息，可以直接删除
      - 考虑是否为用户表达误差，需要进一步澄清
      - 根据上述分析，列出可能的优化目的，询问用户

3. 根据用户确认的待定项结果，更新具体的任务内容到下面的**本次任务内容**中，向用户确认**本次任务内容**是否准确无误，得到确认后再进行下一步，否则根据用户指示进行

4. 按照**本次任务内容**开始完成本次任务

## 异常处理和质量保证
- 如果用户不配合删除待定项，需要再次澄清和确认
- 添加任务完成质量检查标准
- 提供回滚和重新分析的机制
