# React (TypeScript) 项目规范守则套件

本目录聚焦使用 TypeScript 的 React 生态（含 Next.js、Vite 等工具链），提供面向不同交付模式的守则模板。文档整合 React.dev 官方指南、TypeScript 手册、Next.js 生产级实践与设计系统经验，帮助团队构建可维护、可扩展的前端体系。

## 内容总览
- `规范守则_单页应用与组件体系.md`
- `规范守则_SSR与边缘渲染.md`
- `规范守则_设计系统与可访问性.md`
- `规范守则_状态管理与数据同步.md`
- `规范守则_性能优化与质量保障.md`

## 使用建议
1. 默认假设使用 TypeScript + React 18/19 生态，如需兼容其他语言请在项目中补充说明。
2. 结合守则中推荐的目录结构、Lint/格式化配置以及测试策略，建立团队模板或脚手架。
3. 守则引用的资料包括 React.dev、TypeScript 官网、Next.js Production Checklist、TanStack/Redux 指南等，建议定期复查。
4. 如需扩展到 React Native / Electron 等场景，可在对应守则内新增章节或补充文档。
