# 环节核对清单

在每个环节开始或结束时对照以下项目，确认关键动作未遗漏。建议打印或复制到待办工具中逐项打勾。

## 通用
- 已阅读 `Agents.md`、本指引及最新模板，确认任务适用范围。
- `AI-AGENT_working-status.csv` 记录与当前环节一致，且每次更新后均已提交。
- Todo 工具与实际执行同步，关键步骤均有对应提交计划。
- 所有沟通、文档与提交信息使用简体中文。
- 需要时间戳时通过 `mcp-time` 获取。

## 任务前置准备
- `now-task.md` 已完成内容梳理，待定项按模板列出并集中确认。
- `optimized-task.md` 写明优化后的目标、步骤与验证方式。
- 评估是否需要调用 Sequential Thinking，并记录决定理由。

## 流程设计
- 梳理输入、输出、依赖、风险，形成可执行方案或草图。
- 对多方案或疑点列出选项并准备沟通记录。
- `optimized-task.md` 的流程描述与验证标准已经同步更新。

## 任务主体流程循环
- 每进入新子步骤即更新状态记录并提交。
- Todo 与实际执行保持一致，完成后立刻标记。
- 关键决策、风险、返工原因记录在案；流程调整前已同步设计文档。

## 验证结果
- 按 `optimized-task.md` 的验证标准逐项完成检查。
- 单元、集成、静态等必要测试已执行并记录输出要点。
- 验证结论写入状态记录，若失败说明返工计划。

## 文档更新
- 受影响文档（README、CHANGELOG、设计文档等）均已同步。
- 状态记录更新后，先维护 CHANGELOG（写入最新版本与日期，重建 `[Unreleased]`）。
- 模板、流程图、讨论纪要等资料已整理归档。

## 收尾
- 仓库无临时文件或冗余日志，`now-task.md` 恢复为空模板。
- 工作状态记录、Todo、提交计划全部闭环。
- 写入最终状态记录，概述交付成果、遗留风险与后续建议。
