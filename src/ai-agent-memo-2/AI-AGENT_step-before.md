# 任务前置准备操作清单

以下列表按执行顺序编排，可在每项完成后标记，以确保任务在进入主体流程前准备充分。

## 1. 任务分析
- 阅读 `Agents.md` 与本目录指引，确认适用版本与语言要求。
- 通读当前 `now-task.md`，拆解目标、边界与成功标准。
- 评估任务复杂度，若存在多方案或复杂依赖，调用 Sequential Thinking 获取结构化计划。

## 2. 内容优化
- 对 `now-task.md` 中的冗余或歧义条目提出改写建议，并记录潜在风险。
- 使用待定项模板整理需要确认的选项，暂不修改原文。
- 结合历史文档与临时约定，判断是否需额外补充限制或豁免说明。

## 3. 待定项处理
- 将所有待定项集中呈现给用户或决策方，避免零散沟通。
- 根据反馈仅保留采纳方案，删除未被采纳的选项。
- 将确认结果整合进优化后的任务描述，准备进入下一环节。

## 4. 结果生成
- 在 `optimized-task.md` 中写入优化后的任务目标、执行步骤与验证标准。
- 初始化或更新 `AI-AGENT_working-status.csv`，记录当前所处环节与主要结论。
- 准备流程设计所需的输入资料（结构草案、风险清单、依赖列表等），确保可立即开展下一阶段。

> 更多执行细节，请参考本目录中的 `AI-AGENT_criterion.md` 对应章节。
