# Java 项目规范守则套件

本目录聚焦Java生态的多场景工程实践，为AI代理或开发团队提供结构化的规范守则模板。每篇守则结合行业权威资料（如《Effective Java》、Google Java Style、Spring官方手册等），涵盖架构原则、工程结构、工具栈与交付策略，便于快速对齐团队标准。

## 内容总览
- `规范守则_CLI与自动化脚本.md`
- `规范守则_Web与微服务.md`
- `规范守则_企业级平台与服务.md`
- `规范守则_数据工程与大数据.md`
- `规范守则_高性能与分布式计算.md`

## 使用建议
1. 在启动或评估项目时，先确认所属方向，再按文档中的“核心原则”与“测试交付”章节落实实践。
2. 结合企业内部规约（如Google Java Style、阿里巴巴Java开发手册）扩展细则，与CI检查、IDE插件联动执行。
3. 针对框架（Spring Boot、Quarkus、Micronaut、Spark、Flink 等）可将推荐配置写入团队模板或脚手架。
4. 守则中列出的参考资料均为长期维护的权威来源，建议定期复查链接并补充最新版本。
5. 如需新增方向，请同步更新本README与CHANGELOG，保持多语言守则体系的一致性。
